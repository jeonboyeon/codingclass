전역변수와 지역변수

변수는 함수 블록{}을 기준으로 '전역 변수' 와 '지역 변수' 로 나뉩니다.

전역변수

전역변수는 함수 블록{} 밖, 안 어느곳에서든 자유롭게 사용이 가능하다.

지역변수

지역변수는 함수 블록{} 내에서만 사용이 가능하다.

전역변수와 지역변수 선언위치

전역변수 지역변수
var 변수 :
function 함수() {
}		 function 함수() {
var 변수;
}

전역 변수 예시 문제

var kor = 90;
function getScore() {
    kor = 100; //   전역 변수
    console.log(kor);
}
getScore();
console.log(kor);
결과보기
100
함수 블록{} 안에 있는 kor 변수는 전역 변수이다.
따라서 getScore() 함수를 호출하게 되면 kor 변수 값이 90 에서 100으로 바뀌게 된다.

지역 변수 예시 문제

var kor = 90;
    function getScore() {
        var kor = 100; // 지역 변수
        console.log(kor);
    }
    getScore();
    console.log(kor); //전역 변수
결과보기
{} 안의 값은 100 | 지역 변수
{} 밖의 값은 90 | 전역 변수
이번 예시에선 함수 블록{} 안의 kor변수를 var로 정의였는데
이렇게 되면 블록{} 안의 변수는 지역 변수가 되고, 밖의 변수는 전역 변수가 된다.
그렇기 때문에 지역 변수 안에 console.log(kor) 는 닶이 100이 되고,
밖에 있는 console.log(kor) 는 닶이 90이 된다.

이렇게 블록{}에 의해 변수의 범위가 달라지는 것을 변수의 scope 라고 한다.

함수 레벨 스코프와 블록 레벨 스코프

var는 함수 블록{} 에서만 지역 변수가 존재하고 블록{}이나 제어문 블록{} 에서는 지역 변수를 선언하지 못한다.
하지만 ES6 에서 새로 나온 let 과 const 는 블록{}, 제어문 블록{} 에서도 지역 변수를 선언할 수 있다. 

var num = 1;
{
    var num = 10; // num 은 전역 변수
    console.log(num);
}
console.log(num);
결과보기
20

var를 사용 시 블록{} 안에 선언 한 변수를 전역 변수로 인식 


 //-- let 사용 --//
let num = 1;
{
    let num = 10; // num 은 지역 변수
    console.log(num);
}
console.log(num);

//-- const 사용 --//
const num = 1;
{
    const num = 10; // num 은 지역 변수
    console.log(num);
}
console.log(num);
결과보기
블록{} 안의 값은 10
블록{} 밖의 값은 1

let과 const는 제어문 블록{} 안에 선언된 변수를 지역 변수로 인식한다.


연습 문제


//함수의 전역 변수
var sum1 = 10;
function add1() {
    sum1 = 20;
}
add1() {
  console.log('전역' + sum1);
}

//함수의 지역 변수
var sum2 = 30;
function add2() {
    var sum2 = 40;
    console.log('지역' + sum2);
}
add2();
console.log('전역' + sum2);

//블록의 지역 변수
var num1 = 50;
if (num1 === 50) {
    var num1 =60;
}
console.log('전역' + num1);
let num2 = 70;
if (num2 === 70) {
    let num2 = 80;
    console.log('전역' + num2);
}
console.log('전역' + num2);
결과보기

변수 범위 선언 특징
함수 레벨 스코프 var 변수를 중복해서 선언이 가능
함수 블록{} 내에서만 지역변수가 존재
블록 레벨 스코프 let 같은 블록{}에서는 이미 선언한 변수를 중복해서 선언할 수 없다.
블록{}, 제어문 블록{}에서도 지역 변수가 존재
const 같은 블록{}에서 이미 선언한 변수를 중복해서 선언할 수 없고, 저장된 값을 변경할 수도 없다.
